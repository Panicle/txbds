# 智能铁鞋服务端 - 实现计划

## 项目目标
搭建测试用服务端，支持基准站+移动站+NFC的简单场景，部署在服务器上，通过网页展示坐标点

## 技术选型
- 语言: Python 3.10+
- TCP服务端: asyncio
- Web框架: FastAPI + Uvicorn
- 实时通信: WebSocket (推送坐标到前端)
- 前端地图: 自定义SVG地图（无需外部依赖）
- 存储: SQLite (方便部署，支持并发)

## 实现步骤

### Step 1: 协议解析模块 (protocol.py)
实现协议解析器：
- 帧头识别（0x55AA基准站帧, 0xAA55移动站帧）
- 6字节ID解析（设备类型+场区+编号）
- 定位类型识别（0x00人员/0xE1铁鞋初步/0xE2铁鞋改正后/0xE3铁鞋精密）
- 帧完整性校验

### Step 2: 数据存储模块 (storage.py)
- SQLite数据库，两个表：coordinates, nfc_records
- 坐标数据：解析经纬度，按设备ID和时间戳存储
- NFC数据：解析时间戳和NFC号码
- 提供查询接口

### Step 3: 主服务程序 (main.py)
整合TCP服务端与FastAPI：
- TCP服务器，监听两个端口（基准站: 8001，移动站: 8002）
- 使用asyncio处理并发连接
- 差分数据转发（100101→200101）
- WebSocket管理（实时推送坐标）
- 静态文件服务

### Step 4: 前端展示 (web/index.html + web/app.js)
- SVG网格地图（无需外部地图API）
- WebSocket连接接收实时坐标
- 显示位置标记（不同定位类型用不同颜色）
- 实时位置面板
- 历史记录列表

### Step 5: 测试客户端 (test_client.py)
- 模拟基准站发送差分数据
- 模拟移动站发送坐标数据（4种定位类型）
- 模拟移动站发送NFC数据

## 设备ID配置

| 设备类型 | ID | 配对关系 |
|----------|----|----------|
| 基准站A | 100101 | ↔ 移动站A(200101) |
| 基准站B | 100102 | ↔ 移动站B(200102) |

**ID编码规则**（六位数）：
- 第1位：设备类型（1=基准站，2=移动站）
- 第2-3位：场区编号
- 第4-6位：场区内编号

## 帧格式定义

### 基准站差分数据帧
```
| 帧头(2B) | ID(6B) | RTCM数据(NB) | 帧尾(2B) |
| 0x55AA   |        |              | 0xAA55   |
```
- 接收频率：约4条/秒/基准站

### 移动站坐标数据帧
```
| 帧头(2B) | ID(6B) | 定位类型(1B) | 定位数据(20B) | 帧尾(2B) |
| 0xAA55   |        |              |               | 0x55AA   |
```
- 接收频率：约1条/3秒/移动站
- 定位类型：0x00(人员)、0xE1(铁鞋初步)、0xE2(铁鞋改正后)、0xE3(铁鞋精密)

### 移动站NFC数据帧
```
| 帧头(2B) | ID(6B) | 时间戳(10B) | NFC号码(10B) | 帧尾(2B) |
| 0xAA55   |        |             |              | 0x55AA   |
```
- 接收频率：事件触发（读到NFC卡片时）

## 开发顺序
1. protocol.py → 帧解析模块
2. storage.py → SQLite存储
3. main.py → TCP + Web服务整合
4. web/index.html + web/app.js → 前端地图
5. test_client.py → 测试客户端

