# 智能铁鞋服务端 — 实现需求说明书

## 一、系统概述

智能铁鞋定位系统服务端，负责接收基准站和移动站数据，完成差分数据转发、坐标/NFC数据解析存储、铁鞋信息匹配及位置绘图展示。

---

## 二、功能模块

### 模块1：TCP/UDP通信服务

| 需求项 | 说明 |
|--------|------|
| 监听端口 | 服务端需开放端口，接收基准站和移动站的TCP（或UDP）连接 |
| 多连接管理 | 支持多个基准站和移动站同时在线 |
| 连接标识 | 通过帧内6字节ID识别设备身份（区分基准站/移动站） |
| 心跳/断线检测 | 检测设备掉线，清理无效连接 |

### 模块2：协议解析

#### 2.1 ID编码规则（6字节/六位数）

| 位置 | 位数 | 含义 | 值 |
|------|------|------|----|
| 第1位 | 1 | 设备类型 | 1=基准站，2=移动站 |
| 第2-3位 | 2 | 设备所属场区 | 00-99 |
| 第4-6位 | 3 | 场区内编号 | 001-999 |

**示例：**
- 基准站A：`100101`（设备类型1 + 场区00 + 编号101）
- 基准站B：`100102`（设备类型1 + 场区00 + 编号102）
- 移动站A：`200101`（设备类型2 + 场区00 + 编号101）
- 移动站B：`200102`（设备类型2 + 场区00 + 编号102）

#### 2.2 基准站差分数据帧

```
| 帧头(2B) | ID(6B) | RTCM数据(NB) | 帧尾(2B) |
| 0x55AA   | 0x313030313031 | 0xD300... | 0xAA55   |
```

- **帧结构**：帧头(2字节) + ID(6字节) + RTCM数据(N字节) + 帧尾(2字节)
- **接收频率**：约4条/秒/基准站
- **示例帧**：`0x55AA 0x313030313031 0xD300 0xAA55`

#### 2.3 移动站坐标数据帧

```
| 帧头(2B) | ID(6B) | 定位类型(1B) | 定位数据(20B) | 帧尾(2B) |
| 0xAA55   | 0x323030313031 |              |               | 0x55AA   |
```

**定位类型枚举：**

| 值 | 含义 |
|----|------|
| 0x00 | 人员定位 |
| 0xE1 | 铁鞋初步定位 |
| 0xE2 | 铁鞋改正后定位 |
| 0xE3 | 铁鞋精密定位 |

- **帧结构**：帧头(2字节) + ID(6字节) + 定位类型(1字节) + 定位数据(20字节) + 帧尾(2字节)
- **接收频率**：约1条/3秒/移动站

#### 2.4 移动站NFC数据帧

```
| 帧头(2B) | ID(6B) | 时间戳(10B) | NFC号码(10B) | 帧尾(2B) |
| 0xAA55   |        |             |              | 0x55AA   |
```

- **帧结构**：帧头(2字节) + ID(6字节) + 时间戳(10字节) + NFC号码(10字节) + 帧尾(2字节)
- **接收频率**：事件触发（读到NFC卡片时发送）

### 模块3：差分数据转发（基准站→移动站）

| 需求项 | 说明 |
|--------|------|
| ID匹配 | 解析基准站帧中的6字节ID，查找配对的移动站 |
| 配对关系 | **基准站A(100101) ↔ 移动站A(200101)**；**基准站B(100102) ↔ 移动站B(200102)** |
| 转发规则 | 有配对移动站在线 → 原样转发完整帧数据；无配对 → 丢弃不转发 |
| 转发格式 | 转发数据保持接收到的原始格式不变 |

### 模块4：坐标数据处理（移动站数据接收）

| 需求项 | 说明 |
|--------|------|
| 帧识别 | 通过帧头0xAA55 + 定位类型字段区分坐标帧和NFC帧 |
| 数据解析 | 解析6字节ID + 1字节定位类型 + 20字节定位数据 |
| 数据存储 | 坐标数据存储到文件中 |
| 定位类型处理 | 区分人员定位(0x00)与铁鞋定位(0xE1/0xE2/0xE3)，分别处理/存储 |

### 模块5：NFC数据处理

| 需求项 | 说明 |
|--------|------|
| 帧识别 | 通过帧结构区分（无定位类型字段） |
| 数据解析 | 解析6字节ID + 10字节时间戳 + 10字节NFC号码 |
| 铁鞋匹配 | 根据NFC号码匹配铁鞋信息 |
| 数据存储 | NFC刷卡记录持久化存储 |

### 模块6：绘图展示

| 需求项 | 说明 |
|--------|------|
| 触发方式 | 用户点击"绘图"按钮 |
| 展示内容 | 当前移动站的位置 + 铁鞋的位置 |
| 地图/画布 | Web前端地图展示 |
| 实时性 | 展示最新位置信息 |

## 三、数据模型

### 3.1 设备信息

```
设备ID: 6字节（唯一标识）
  ├─ 设备类型: 1字节（1=基准站, 2=移动站）
  ├─ 所属场区: 2字节
  └─ 场区编号: 3字节

配对关系: 基准站ID → 移动站ID（一对一对一）
  ├─ 100101 → 200101
  └─ 100102 → 200102
```

### 3.2 坐标记录

```
移动站ID: 6字节
定位类型: 1字节（0x00/0xE1/0xE2/0xE3）
定位数据: 20字节
接收时间: 服务器时间戳
```

### 3.3 NFC记录

```
移动站ID: 6字节
时间戳: 10字节（设备端时间）
NFC号码: 10字节
匹配铁鞋: （由NFC号码查映射表得到）
```

## 四、关键技术决策点（需确认）

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| 1 | 通信协议用TCP还是UDP？ | 差分数据4条/秒需可靠传输 | TCP更可靠；若考虑延迟可用UDP+重传 |
| 2 | 定位数据20字节的具体编码格式？ | 决定坐标解析逻辑 | 需与硬件方确认，可能是IEEE754浮点或定点数 |
| 3 | 坐标数据存储格式？ | 影响后续查询和绘图 | 建议SQLite，按设备和时间戳组织 |
| 4 | NFC号码与铁鞋的映射规则？ | 核心业务逻辑 | 需确认：固定映射表？还是动态注册？ |
| 5 | 设备配对关系如何管理？ | 运维灵活性 | 当前硬编码配对关系(100101↔200101, 100102↔200102) |
| 6 | 绘图方案：Web前端还是桌面端？ | 技术栈选择 | 建议Web方案（前后端分离），便于多终端访问 |
| 7 | NFC帧与坐标帧如何区分？ | 协议解析 | 通过帧长度或定位类型字段区分 |

## 五、系统架构

```
┌─────────────┐  TCP   ┌──────────────────────────────────────┐
│  基准站 A/B  │ ─────→ │  通信层（TCP Server, 端口监听）        │
└─────────────┘        │     ├─ 连接管理（多设备在线）            │
                       │     ├─ 帧解析（4种帧格式）              │
                       │     └─ 帧分发（按类型路由）             │
┌─────────────┐  TCP   │                                      │
│  移动站 A/B  │ ─────→ │  业务层                                │
└─────────────┘        │     ├─ 差分转发（基准站→配对移动站）     │
                       │     ├─ 坐标处理（解析+存储）            │
                       │     ├─ NFC处理（解析+铁鞋匹配）         │
                       │     └─ 配对管理（ID映射表）             │
                       │                                      │
                       │  数据层                                │
                       │     ├─ SQLite存储（坐标/NFC数据）       │
                       │     ├─ 映射表（NFC↔铁鞋）              │
                       │     └─ 配对关系（硬编码）                │
                       │                                      │
                       │  展示层                                │
                       │     └─ Web API + WebSocket实时推送      │
                       │     └─ 前端地图展示                    │
                       └──────────────────────────────────────┘
```

## 六、性能指标估算

| 指标 | 估算值 | 说明 |
|------|--------|------|
| 基准站数据吞吐 | ~4帧/秒/站 | 差分数据 |
| 移动站坐标数据 | ~0.33帧/秒/站 | 1帧/3秒 |
| 移动站NFC数据 | 事件触发 | 读卡时发送 |
| 并发连接数 | 基准站+移动站总数 | 初期4台 |
